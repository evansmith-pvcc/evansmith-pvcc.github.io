Chapter 7: Software Engineering Terms

Term Definition
Software Engineering The application of engineering principles to software development in a systematic method.
Integrated Development Environment (IDE) An application that provides comprehensive facilities to computer programmers for software development.
Association for Computing Machinery (ACM) A global scientific and educational organization dedicated to advancing the art, science, engineering, and application of computing.
Purpose of ACM Advances the computing as a science and a profession.
Benefits of ACM Membership Access to publications, courses, and new networking opportunities.
Student ACM Membership Special pricing for students, including access to a vast library of resources.
Institute of Electrical and Electronics Engineers (IEEE) A professional association for electronic engineering and electrical engineering (and associated disciplines).
Purpose of IEEE To foster technological innovation and excellence for the benefit of humanity.
Benefits of IEEE Membership Access to the industry's most essential technical information, networking opportunities, career development tools, and several savings opportunities.
Student IEEE Membership Discounted membership rates for students, including digital library access and opportunities to connect with industry professionals.
Software Life Cycle: Development Phase Describes the process of developing software through successive phases in an orderly way.
Requirements Analysis Defines project goals into defined functions and operation of the intended application.
Design Process of envisioning and defining software solutions to one or more sets of problems.
Implementation The part of the process where programmers actually program the code for the project.
Testing Stage where the software is tested for defects and ensure that it operates as expected.
Modular Implementation A method that involves breaking down a project into smaller, manageable parts or modules that are developed independently.
Unified Modeling Language (UML) A standardized modeling language enabling developers to specify, visualize, construct and document artifacts of software systems.
Use Case A description of a system’s behavior as it responds to a request that originates from outside of that system.
Use Case Diagram A diagram that represents a user's interaction with the system and outlines the specifications of a use case.
Data Relationships: Defines how data is connected and how each part of the database is related to one another.
One-to-One A relationship where one record in a table may have only one matching record in another table, and vice versa.
One-to-Many A relationship where a single record in one table can be associated with one or more records in another table.
Many-to-Many A relationship where multiple records in one table are associated with multiple records in another table.
Design Pattern Typical solutions to common problems in software design, each pattern being a specific arrangement of objects and classes that solve a particular problem.
Software Quality Assurance (SQA) A process which ensures that developed software meets and complies with defined or standardized quality specifications.
Software Testing The process of evaluating and verifying that a software product or application does what it is supposed to do.
Glass Box Testing Testing an application with knowledge of the internal workings of the item being tested.
Black Box Testing Testing where the software tester does not know the internal workings of the application being tested.
Beta Testing A testing period where a sample of the intended audience tries the product out.
Alpha Testing Testing of an early version of software performed by users who are typically internal employees of the organization developing the software.
Documentation The written detailed information about the software's design, functionality, and usage.
User Documentation Guides and manuals for the end-users to help them understand and effectively use the software product.
System Documentation Detailed descriptions of the workings of a system from the technical standpoint, used by system administrators and other IT professionals.
Technical Documentation Documentation that describes handling, functionality and architecture of a technical product or a product under development or use.
Intellectual Property Law Laws that secure and enforce legal rights to inventions, designs, and artistic works.
Software License The legally binding agreement that specifies the terms of use and distribution of software.
Software Disclaimer A statement to specify or limit the scope of rights and obligations that may be exercised and enforced by parties in a legally recognized relationship.
Best Defenses Against Software Liability Claims Apply sound software engineering principles during the software’s development, to use a level of care compatible with the software’s application, and to produce and maintain records that validate these endeavors.
