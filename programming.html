Chapter 5: Programming Algorithms & Software Testing

Phases of computer program development:

  1. Understand the problem
  2. Design an algorithm to solve the problem
  3. Write the program code in appropriate language based on the algorithm
  4. Enter the program and save
  5. Execute the program
  6. Test the program, find errors and debug, then retest
  7. Document and evaluate the program

Programming structures:

  • Sequential structure: Top-down execution
  • Decision structures: if/then/else
  • Iterative (Looping):
    • while loop: continue looping as long something is true
    • repeat/until loop: continue looping until something is true
    • for loop: loop a given number of times
  • Recursive structures:
    • Repeat a set of instructions as a subtask of itself
    • Repetition continues until: Degenerative case, which is a termination condition, is met (degenerative case is also called a "base case")

Two classic programmng/database problems

  • Sorting: Descending order/Ascending order
    • Sorting numbers
    • Sorting strings (in accordance with ASCII values)
  • Searching: find specific data that meet a certain criteria
    • In database:
    • Filters: temporary search
    • Queries: search criteria are saved so they don't have to be recreated
  • Two popular search techniques:
    • Sequential search:
    • search from top to bottom or bottom to top, one item at a time
    • Binary search:
    • only works if data is SORTED first start in the middle, then go to half of that, etc.

Two classic program goals:

  • Efficiency: Program takes minimal time and/or resources
  • Correctness: Program runs in expected ways and works for all expected conditions

Software testing and verification

  • Kinds of tests:
    • Acceptance testing:
    • Ensures the system's readiness for real-world use.
    • Integration testing:
    • Focuses on determining the correctness of interactions between integrated units.
    • Unit testing:
    • Aims at isolating each part of the program and showing that individual parts are correct.
    • Functional testing:
    • Tests the functionalities of the software system, including user interfaces, APIs, databases, security, and all other functional aspects.
    • Performance testing:
    • Ensures the software applications will perform well under their expected workload.
    • Regression testing:
    • Re-running functional and non-functional tests to ensure previously developed and tested software still performs after a change.
    • Stress testing:
    • Tests the system's ability to handle extreme workloads and how it recovers from failure.
    • Usability testing:
    • Measures the usability, or ease of use, of a specific object or set of objects.
  • Software testing is done to detect:
    • Architectural flaws:
    • Identifies fundamental problems with the system's architecture that could lead to major issues down the line.
    • Poor design decisions:
    • Highlights decisions that could negatively impact the system's effectiveness or maintainability.
    • Invalid or incorrect functionality:
    • Checks for functions that fail to perform their intended tasks correctly.
    • Security vulnerabilities:
    • Identifies potential threats to the software's security, aiming to protect data and prevent breaches.
    • Scalability issues:
    • Assesses the software's capacity to expand and accommodate growth without performance degradation.
  • Testing best practices:
    • Continuous testing:
    • A practice of executing automated tests as part of the software delivery pipeline to obtain immediate feedback on the business risks associated with a software release candidate.
    • Configuration management:
    • Involves the identification, documentation, and control of changes to the software's configuration.
    • Service virtualization:
    • Enables testing of the application even when some components or services are not available for testing.
    • Defect or "bug tracking":
    • A systematic approach to finding, documenting, and managing issues within the software.
    • Metrics and reporting:
    • Involves collecting, analyzing, and reporting data to enhance decision-making processes and improve software quality.
